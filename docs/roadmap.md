# Roadmap

The first release focuses on `package.json` dependency health scoring and professional report output. Future versions can expand into deeper supply-chain analysis.

## Planned Features

- GitHub Action integration
- Lockfile analysis
- npm audit integration
- SBOM generation
- Package replacement suggestions
- Web dashboard
- Monorepo and workspaces support
- Trusted publishing and provenance checks
- Dependency diff between branches
- Organization-level dependency reports

## Near-Term Improvements

- Add registry metadata fixtures for parser tests
- Add package score configuration options
- Add markdown report output
- Add severity thresholds for exit codes
- Add examples for common CI providers

## Longer-Term Ideas

- Compare dependency health between release branches
- Detect packages with sudden maintainer changes
- Add package age and download trend context
- Create reusable report schemas for integrations
- Support policy files for organization-specific rules
